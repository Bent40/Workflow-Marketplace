# writer-help — decision log

## Session 2026-07-06 (headless build, branch claude/literary-workflow-feedback-cdlwlt)

- **Origin**: explicitly user-requested ("The concept of BMAD-help, but for
  packs. If i go writer-help, it should look at what the project currently
  holds, the cannon, the in progress, and help guide me on which tools would
  help in what situation.").
- **Governance**: CREATE-NEW, project-specific, member of the `writer`
  install pack. bmad-help is the proven pattern but is BMAD-lifecycle-bound
  and an upstream soft-fork file — extending it to probe literary state
  (canon store, style profiles, revision chains) would tangle upstream logic.
  Boundary: writer-mode loads the POSTURE (static routing); writer-help reads
  the SITUATION (state probe → stage inference → few recommendations).
- **Pack-help pattern recorded** in docs/literary-pack.md as replicable
  (`<pack>-help` = state probe + stage inference + right-sized
  recommendations with invoking phrases); other packs adopt on demand, not
  speculatively.
- **Design calls**: inventory-not-contents probe (privacy + context
  discipline — count files and facts, never read prose); max three
  recommendations + exactly one housekeeping flag (guidance, not a process
  checklist); honest-roadmap rule (never recommend unbuilt skills — name
  today's substitute); direct-question mode skips the probe.
- **Companion change (not a skill)**: writer-mode's harvest section gains
  bounded per-task TRACE capture for the pack's calibration period (user
  decision 2026-07-06): one 2–4 line trace per completed task (ask → route →
  outcome → friction) appended to _workflow/writer-mode-feedback.md; never
  mid-flow, never per-message; revisit when the pack stabilizes.
- **Validation mode**: degraded in-session (trigger proxy + 3 artifact
  sims); hermetic re-run via bmad-eval-runner when available.
- **Validation results (2026-07-06)**: trigger proxy 9/9 ("bmad help" and
  sprint queries route to bmad-help; "help me brainstorm" routes direct to
  brainstorming — help is not a toll booth). Artifact evals 3/3 PASS —
  wh-1 revision-chain (stage inferred, ≤3 moves with phrases, exactly one
  housekeeping flag via wf learn not unbuilt lit-canon, undeclared game
  surfaced, past misroute from the feedback file reused as a warning);
  wh-2 empty-project (ideation-only, zero ceremony); wh-3 direct-question
  (no probe, honest no-pacing-analyzer routing to structural review).
