# 3d-create — decision log

## Session 2026-07-07 (branch claude/3d-pack-integration-eutnkd)

- **Origin**: user-requested, same effort — this is the "invokable pipeline"
  the `wf learn` note (`3d-modeling-pack-drafted`) said was the one thing that
  needed building. It is the pack's doing-skill (analogue of the `lit-*` craft
  skills under the writer pack).
- **Governance**: CREATE-NEW under the 3D-pack verdict. Distinct from
  `3d-mode` (posture) and `3d-help` (routing/guide): this carries the
  step-by-step pipeline judgment and the trap-avoidance.
- **Name**: `3d-create` (the create/build verb in the pack namespace).
- **Classification**: Simple Workflow, fully inline (~95 lines). Deliberately
  **references `docs/3d-modeling/` for exact numbers** rather than duplicating
  them — governance: knowledge stays in docs, the skill carries only the
  judgment an LLM would otherwise get wrong.
- **Content bar — the six traps that justify the skill** (each cost real hours
  on the box, none an LLM avoids unprompted): (1) output→engine routing
  (print/game/minecraft → Hunyuan/Hybrid/TRELLIS); (2) clean-before-generate /
  decapitation; (3) watertight-verify-never-assert (bmesh + backface render,
  never trimesh); (4) multiview sliver-collapse (own model + turnaround only);
  (5) VRAM: shape and texture are separate stages; (6) sequential-Blender +
  the no-pip/local hard constraints. The Y-up/Z-up render gotcha is included
  because a lying-down render misreads as failure.
- **Trigger design**: "turn this image into a print/game asset/Minecraft build",
  "make a 3D model from this art", "print this character", "voxelize this".
  Negative boundary: data-model, code generation, and ML-model training must
  not fire it.
- **Validation mode**: degraded in-session (as the sibling skills); re-run with
  bmad-eval-runner when Docker + key are available.
- **Escort-strengthening pass (2026-07-07)**: added a "How to escort" section
  (one step at a time, confirm before each consequential run, never batch past a
  gate) and a **human-review checkpoint** (`bmad-checkpoint-preview`) at the
  watertight gate — grafting BMAD's best escort primitive onto the light
  skill-pack, per the user's "escort the user through creation" goal.
- **Validation result (2026-07-07, degraded)**: trigger sanity **8/8 match**,
  artifact evals **6/6 PASS** via a fresh validator subagent. The validator
  flagged the original 5 evals as recall-aligned, so a distractor
  (`mc-6-watertight-and-gate-distractor`) was added — it tests that the
  two-part watertight AND-gate is *enforced* (render-alone ≠ done) and routes
  through the checkpoint; it PASSes against the strengthened Step 5.
