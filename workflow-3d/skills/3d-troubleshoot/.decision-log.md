# 3d-troubleshoot — decision log

## Session 2026-07-07 (branch claude/3d-pack-integration-eutnkd)

- **Origin**: user-approved roadmap skill ("go ahead with the roadmap skills").
  Governance-approved-but-unbuilt in the 3D pack charter; built this session.
- **Governance**: CREATE-NEW, greenlit in the charter roadmap. Distinct lane:
  `3d-create` runs the happy path (and gates it), `3d-setup` gets the box ready,
  this one is the **post-failure diagnostic** — symptom → cause → fix for a run
  that already went wrong. No overlap.
- **Name**: `3d-troubleshoot` (pack prefix; the roadmap name already advertised
  by the charter and `3d-help`).
- **Classification**: Simple Workflow, fully inline (~85 lines). The value is a
  compact **symptom→cause→fix table** encoding failures that each cost hours and
  are non-obvious (a headless torso reads as a model bug, a lying-down render
  reads as an empty result, a trimesh "watertight" reads as success). Exact
  numbers stay in `rules.md`/`settings.md`/`hardware-scouting.md`/multiview.
- **Content bar**: the 11 diagnoses are the hard-won ones — decapitation crop,
  the #1 sliver-collapse trap, trimesh false-positive, floating flakes, OOM
  staging, Y-up/Z-up, parallel-Blender corruption, RAM-offload slow-decode
  (not-hung), TRELLIS detail ceiling, sdpa backend, Blackwell cu128. Enforces
  the watertight-honesty rule and "detail comes from the generator, not the
  remesh".
- **Trigger design**: "3d troubleshoot", "my print has holes", "mesh came out
  mangled/tiny", "it OOMed", "render is empty/lying down", "why did generation
  fail". Negatives: unit tests, API 500s, DB slowness must not fire it.
- **Hard rule carried**: never propose a `pip install` into the ComfyUI `.venv`
  as a fix — the fix is inputs/settings/sequencing/box, not the venv.
- **Validation mode**: degraded in-session (no Docker/API key in-container);
  re-run hermetically with bmad-eval-runner when available.
- **Validation result (2026-07-07, degraded)**: trigger sanity **8/8 match**,
